## Methodology confirmed from the prior session

- Model outputs were generated externally by the operator with
  `claude -p --model haiku`.
- Tools were disabled for the model run, so the model performed its own
  no-tool engineering calculation.
- Every run was captured, including passes and failures.
- Raw output bytes were stored verbatim under `captures/runs/`.
- Each capture carries a `source.json` record with prompt hash, raw-output hash,
  model metadata, run date, and artifact paths.
- Promoted benchmark cases reference the captured artifact hashes rather than
  reviewer-written reconstructions.
